Embedding Layers & Shared Models (Keras)

Keras models can have both trainable and fixed parameters. Fewer trainable parameters are less flexibel but at the same time less likely to overfit. Trainable parameters are usually in dense layers. Due to an embedding layer in the model (input) and dense layer (with e.g. 4 parameters), the model has much more trainable parameters. Embedding layers often add a large number of trainable parameters into the model (be cautious of overfitting).

Shared Layers (Keras)

Learned about shared layers allowed by Keras API. They allow me to define an operation and then apply the exact same operation, with the exact same weights on different inputs. Can by used for time-series as well. I understand, defining multiple input layers for multiple entities (e.g. customer IDs) allows me to specify how the data from each entity will be used differently later in this model. Now, I see an applied case in UFC.

Activation Function ReLu

Activation function: Inputs (variables or outputs of neurons) get weights. If the dot product of these weights is higher than certain value, the activation function simulata if the neuros fire or not. Activation function is close to a sigmoid function (e.g. square root function), so the output is between 0 and 1. Activation function captures non-linearities. Classification: Dog (0.71) or cat(0.29), i.e. 71% confidence the object is a dog. So why to use DL instead of linear regression?
